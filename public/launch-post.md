# Launch: CI Failure Triage Kit

CI failures waste engineering time because the first responder has to read a noisy log, infer the owner, and write a useful handoff from scratch.

CI Failure Triage Kit is a small local browser app that turns pasted CI logs into a Markdown report with likely cause, owner, next action, and high-signal excerpts.

It is deterministic and local-only. Logs are not uploaded.

Useful for:

- platform engineers
- startup CTOs
- maintainers
- teams without a dedicated CI owner

Public samples:

- Runner OOM: `https://github.com/agentydragon/ci-failure-triage-kit/blob/main/public/samples/oom-report.md`
- Registry 503: `https://github.com/agentydragon/ci-failure-triage-kit/blob/main/public/samples/network-report.md`
